OpenLM Components

Download the necessary components to enable full functionality of your OpenLM system. Each component plays a critical role in license monitoring, usage tracking, and user management.

Broker

v25.4.11.1409LinuxWindows

The OpenLM Broker is installed on the license server machine and facilitates communication between the OpenLM Server and license managers.


Learn more →

Workstation Agent

v25.4.18LinuxWindows

The OpenLM Workstation Agent is a lightweight application that runs on the user's workstation. It captures real usage data and idle times. Furthermore, it identifies idle open sessions and safely releases them. Install it on any number of workstations.


Learn more →

Workstation Agent – Side-by-Side Mode

v25.4.18LinuxWindows

Run alongside Legacy OpenLM and Annapurna.

Note:
For silent installation, ensure your script includes:SETUP_MODE="Standalone".
Without this setting, the Workstation Agent (side-by-side) will uninstall the existing product. When configured correctly, it installs it side-by-syde.


Learn more →

Directory Synchronization Agent

v1.25.318Windows

The DSA works with the Directory Synchronization Service (DSS) by retrieving sync definitions, collecting user data from domain directories, and sending it back to DSS. It can be installed on any machine in your network.
• For local directories (e.g., LDAP), the machine must have network access to the directory.
• For cloud directories (e.g., Azure AD, Google), the machine must have external internet access to reach the directory.


Learn more →

Installation Instructions

Each component has specific installation requirements. For detailed instructions, please refer to our documentation.

  • Download the appropriate component installer for your system.
  • Run the installer as an administrator on the target machine.
  • Follow the on-screen instructions to complete the installation.
  • Configure the component to connect to your OpenLM Server.
